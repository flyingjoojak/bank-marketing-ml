# 은행 마케팅 데이터 기반 예금 가입 여부 예측

## 1. 문제 정의
은행 마케팅 캠페인 데이터를 활용하여  
고객이 **정기예금에 가입할지 여부(y)** 를 예측하는 이진 분류 문제를 해결한다.

본 프로젝트의 목적은  
- 고객 특성에 따른 가입 여부 패턴을 분석하고
- 여러 분류 모델을 비교하여
- 가장 합리적인 모델 선택 기준을 도출하는 것이다.

---

## 2. 데이터 설명
- 데이터 출처: UCI Bank Marketing Dataset
- 파일명: `bank-additional-full.csv`
- 데이터 크기: 약 41,000건
- 타겟 변수: `y` (정기예금 가입 여부)

### 주요 특징
- 범주형 변수: 직업, 결혼 여부, 교육 수준, 연락 수단 등
- 수치형 변수: 나이, 이전 연락 횟수, 소비자 지표 등
- 마케팅 이력 및 경제 지표가 함께 포함된 실제 금융 데이터

---

## 3. 데이터 탐색 및 전처리

### 3.1 데이터 탐색 (EDA)
- `describe(include="all")`을 활용해 전체 분포 확인
- 타겟 변수(y)의 클래스 분포 확인
- 수치형/범주형 변수 혼합 구조 확인

### 3.2 전처리 과정
- 타겟 변수 `y`를 이진 레이블로 변환
- 범주형 변수는 Label Encoding 적용
- 학습 안정성을 위해 **StandardScaler로 수치 데이터 정규화**
- `train_test_split` 사용
  - test size: 20%
  - stratify 적용으로 클래스 비율 유지

---

## 4. 모델링 전략

### 사용한 모델
본 프로젝트에서는 **모델 복잡도 증가에 따른 성능 변화를 비교**하기 위해
아래 세 가지 모델을 선택하였다.

1. **Logistic Regression**
   - 선형 모델 기반의 baseline
   - 해석 가능성이 높아 기준 성능 확인용

2. **Decision Tree**
   - 비선형 패턴 학습 가능
   - 과적합 가능성 관찰 목적

3. **Random Forest**
   - 앙상블 기반 모델
   - 단일 트리 대비 일반화 성능 기대

---

## 5. 학습 및 평가 방법

### 평가 지표
- Confusion Matrix
- Precision / Recall / F1-score
- Classification Report 전체 비교

### 학습 방식
- 동일한 train/test split 사용
- 동일한 스케일링 데이터 적용
- 모델 간 성능을 공정하게 비교

---

## 6. 결과 요약

- Logistic Regression은 안정적인 baseline 성능을 보였으나
  비선형 패턴 학습에는 한계가 있었다.
- Decision Tree는 특정 클래스에 대해 성능 편차가 크게 나타났으며
  과적합 가능성이 관찰되었다.
- Random Forest는 전반적으로 가장 균형 잡힌 성능을 보였으며,
  Precision과 Recall 모두에서 개선된 결과를 확인할 수 있었다.

➡ 단일 모델보다는 **앙상블 기반 접근이 해당 데이터셋에 더 적합**함을 확인하였다.

---

## 7. 한계점 및 향후 개선 방향

### 한계점
- 하이퍼파라미터 튜닝을 제한적으로만 수행
- 클래스 불균형 문제에 대한 별도 처리 미적용
- Feature Importance에 대한 심층 분석 미흡

### 향후 개선
- GridSearch / RandomSearch 기반 하이퍼파라미터 최적화
- SMOTE 등 클래스 불균형 보정 기법 적용
- Feature Importance 및 SHAP 기반 해석 추가
- XGBoost, LightGBM 등 Gradient Boosting 모델 비교

---

데이터 특성과 모델 복잡도 간의 관계를 이해하고
